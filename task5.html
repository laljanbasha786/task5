HTML Interview Questions

HTML stands for HyperText Markup Language and is the language of the internet. It is the standard text formatting language used for creating and displaying pages on the Internet. HTML documents are made up of the elements and the tags that format it for proper display on pages.

HTML tags are used for placing the elements in the proper and appropriate format. Tags use the symbols < and > to set them apart from the HTML content. The HTML tags need not be closed always. For example, in the case of images, the closing tags are not required as in <img> tag.

Attributes are the properties that can be added to an HTML tag. These attributes change the way the tag behaves or is displayed. For example, an <img> tag has an src attribute, which you use to add the source from which the image should be displayed. Attributes are added right after the name of the HTML tag, inside the brackets, and can only be added to opening or self-closing tags, never to closing tags.

The marquee is used for scrolling text on a web page. It scrolls the image or text up, down, left, or right automatically. To apply a marquee, you have to use <marquee> tags.

We separate a section of texts in HTML using the following tags:
  • <br> tag – It is used to separate the line of text. It breaks the current line and shifts the flow of the text to a new line.
  • <p> tag – This tag is used to write a paragraph of text.
  • <blockquote> tag – This tag is used to define large quoted sections.

The list types in HTML are:
  • Ordered list – Uses <ol> tag and displays elements in a numbered format.
  • Unordered list – Uses <ul> tag and displays elements in a bulleted format.
  • Definition list – Uses <dl>, <dt>, and <dd> tags and displays elements in a definition form like in a dictionary.

We can align the list elements in an HTML file by using indents. If you indent each nested list further than the parent list, you can easily align and determine the various lists and the elements that they contain.

An unordered list uses <ul> and </ul> tags, with each element written between <li> and </li> tags. The list items are displayed as bullets rather than numbers. An ordered list uses <ol> and </ol> tags, with each element written between <li> and </li> tags. The list items are displayed as numbers rather than bullet points.

An element in HTML is a set of tags that define a specific part of a web page. It consists of a start tag, content, and an end tag.

Collapsing white space in HTML can help to reduce the size of web pages and make them load faster. It involves removing unnecessary white space between HTML elements.

HTML Entities are special characters used to represent characters that cannot be typed on a keyboard. They are often used to display special symbols and foreign characters.

CSS Interview Questions

CSS (Cascading Style Sheets) is a stylesheet language used to style the appearance of HTML elements on a web page.

CSS allows for separation of presentation and content, makes it easier to maintain consistent styles across a website, and makes it easier to update the look and feel of a website.

CSS rules are written in the following syntax: selector { property: value; }.

A selector is used to select one or more HTML elements on a web page that you want to apply styles to.

A property is a styling option that can be applied to an HTML element, such as color, font-size, or background-image.

A value is a specific setting for a CSS property, such as red for the color property or 16px for the font-size property.

You can link a CSS file to an HTML file using the <link> element in the head section of the HTML file, like this: <link rel="stylesheet" href="styles.css">

The three ways to add CSS to an HTML file are: Inline CSS, Internal CSS, and External CSS.

An ID selector is used to select a single HTML element with a specific ID attribute, like this: #my-id {}.

A class selector is used to select one or more HTML elements with a specific class attribute, like this: .my-class {}.

An ID selector selects a single element with a specific ID attribute, while a class selector selects one or more elements with a specific class attribute.

A pseudo-class is used to apply styles to an element based on a specific state or condition, such as when the element is hovered over or clicked on.

The box model is a way of understanding how CSS styles are applied to HTML elements. It consists of the content area, padding, border, and margin.

The display property determines how an HTML element should be displayed on a web page. Examples include block, inline, inline-block, and none.

display: block causes an element to take up the full width of its container and start on a new line, while display: inline-block allows an element to be inline with other elements but also allows for setting of width and height.

The position property is used to specify how an HTML element should be positioned on a web page. Examples include static, relative, absolute, and fixed.

position: relative positions an element relative to its normal position on the page, while position: absolute positions an element relative to its nearest positioned ancestor.

The float property is used to position an HTML element to the left or right of its container.

The clear property specifies what elements can float next to a particular element and on which side(s).

You can center an element horizontally by setting its margin-left and margin-right properties to auto, and center it vertically by setting its display property to flex and using the align-items and justify-content properties set to center.

You can create a responsive design using media queries, which apply different styles based on the screen size and orientation of the device. You can also use flexible units like percentages and em/rem to make elements scale proportionally with the viewport size.

You can create a hover effect by using the `:hover` pseudo-class and applying styles to the element when it is hovered over, such as changing the background color or adding a border.

You can create a dropdown menu using the `:hover` or `:focus` pseudo-classes on a parent element and applying styles to child elements when the parent is hovered over or focused on.

You can create a responsive navigation menu using media queries to adjust the layout and styling based on screen size. For smaller screens, you can use a hamburger icon and dropdown menu.

You can create a gradient background using the `linear-gradient()` or `radial-gradient()` functions in CSS. For example, `background: linear-gradient(to right, red, blue);` creates a horizontal gradient from red to blue.

You can create a sticky footer by using `min-height: 100vh` on the container and positioning the footer with `position: fixed` and `bottom: 0`.

You can create a responsive grid layout by using `display: grid` on a container and defining columns with `grid-template-columns`. Use media queries to adjust the layout for different screen sizes.

Git & Github Interview Questions

Git is a version control system that helps track changes in source code during software development. GitHub is a web-based platform that uses Git for version control and provides collaboration features such as issue tracking, code reviews, and project management.

One of the most used Git commands is `git status`, which shows the state of the working directory and staging area, including changes that are staged, unstaged, and untracked.

To change branches in Git, you use the `git checkout` command followed by the branch name. For example: `git checkout branch-name`.

The `config` in Git is used to set configuration options for the repository or user. It includes settings like user name, email, and other preferences. GitHub-specific configurations can be set using Git commands like `git config`.

In Git, "staged" refers to changes that have been added to the staging area using the `git add` command, which means they are ready to be included in the next commit.

The `git init` command initializes a new Git repository in the current directory, creating the necessary metadata and directory structure for version control.

The `git clone` command creates a copy of an existing Git repository, including its history, branches, and commits, in a new directory.

The `git add` command adds changes in the working directory to the staging area, preparing them to be included in the next commit.

The `git commit` command creates a new commit with the changes that have been staged using `git add`, saving them to the repository's history.

The `git status` command displays the current state of the working directory and staging area, showing changes that are staged, unstaged, or untracked.

The `git diff` command shows the differences between commits, the working directory, and the staging area, highlighting changes made to files.

The `git log` command shows the commit history for the repository, including details such as commit hashes, authors, dates, and commit messages.

The `git branch` command lists, creates, or deletes branches in the repository. It also indicates the currently checked-out branch.

The `git checkout` command switches to a different branch or restores files in the working directory to their state at a particular commit.

The `git merge` command combines changes from different branches into the current branch, integrating the histories of both branches.

The `git pull` command fetches changes from a remote repository and integrates them into the current branch, combining `git fetch` and `git merge`.

The `git push` command updates the remote repository with changes from the local repository, sending commits and updates to remote branches.

The `git remote` command manages a set of tracked repositories, allowing you to add, remove, or view information about remote repositories.

The `git fetch` command downloads objects and references from another repository, updating the local repository with new commits and branches without merging them.

The `git reset` command resets the current HEAD to a specified state, which can be used to undo changes or move the branch pointer.

The `git revert` command creates a new commit that undoes the changes made by a previous commit, effectively "reverting" those changes.

The `git tag` command creates, lists, deletes, or verifies tag objects that are used to mark specific points in the repository's history, often for releases.

The `git stash` command temporarily shelves changes in the working directory that are not yet ready to be committed, allowing you to switch branches or work on other tasks.

The `git submodule` command manages nested Git repositories within a parent repository, allowing you to initialize, update, or inspect these submodules.

The `git rm` command removes files from both the working tree and the staging area, deleting them from the repository and preparing for their removal in the next commit.
